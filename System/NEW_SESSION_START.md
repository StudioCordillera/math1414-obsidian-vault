## ðŸ“š ESSENTIAL METHODOLOGIES

### Universal Knowledge Framework
Every textbook extraction includes:
1. **Explicit Knowledge**: Definitions, formulas, procedures (stated directly)
2. **Implicit Knowledge**: Prerequisites, assumptions, unstated steps
3. **Common Errors**: What NOT to do (6-category taxonomy)
4. **Relational Connections**: How concepts relate across mathematics

### Node Standard (FOR CONCEPT NODES)
Every atomic concept must have:
```yaml
type: [Topic|Definition|Claim|Example|Method|Question]
status: [draft|in-progress|review|stable]
importance: [low|medium|high|critical]
tags: [controlled vocabulary]
sources: [cite-keys]
source_refs: ["Source Â§Section p.Page"]
relations:
  broader: []
  narrower: []
  depends_on: []
  defines: []
  related: []
  used_in: []
```

### Extraction Process
For textbook chapters:
1. Read section from textbook PDF
2. Extract methods with step-by-step procedures
3. Document explicit and implicit knowledge
4. Identify common errors for each method
5. Note cross-chapter connections
6. Include worked examples with full solutions
7. Create relational framework (vertical & horizontal)

### Concept Node Creation
For atomic concepts:
1. Create separate `.md` file in appropriate location
2. Follow Node Standard structure
3. Include: definition, examples, applications, misconceptions
4. Add 5-6 relational connections (broader/narrower/depends/related/used_in)
5. Write 200-250 lines comprehensive documentation
6. Link bidirectionally to related concepts
